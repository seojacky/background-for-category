Устанавливает фоновые изображения для рубрик верхнего уровня WordPress.

== Description ==

WP Booster: Background for Category — плагин для автоматической установки фонового изображения страницы в зависимости от рубрики верхнего уровня.

Изображения подключаются по шаблону имени файла на основе slug рубрики. Например, для рубрики со slug `wot` плагин подключит файл `background-wot.jpg`.

**Требования:**
* Функция или плагин `get_top_term()` должны быть установлены отдельно.
* Изображения должны находиться в папке `/images/backgrounds/` в корне сайта.
* Файлы именуются по шаблону: `background-{slug}.jpg`.
* Для главной страницы используется файл `background-home.jpg`.

**Возможности:**
* Автоматическая подстановка фона по slug рубрики верхнего уровня.
* Настраиваемый дефолтный цвет фона (используется если изображение не найдено).
* Страница настроек в разделе «Параметры» админ-панели.

== Installation ==

1. Загрузите папку плагина в `/wp-content/plugins/`.
2. Активируйте плагин через меню «Плагины» в WordPress.
3. Убедитесь, что функция `get_top_term()` доступна на вашем сайте.
4. Разместите изображения фонов в `/images/backgrounds/` в корне сайта.
5. Перейдите в «Параметры → Фон категории» и укажите дефолтный цвет фона.

== Frequently Asked Questions ==

= Где должны лежать изображения? =
В папке `/images/backgrounds/` в корне сайта (рядом с `wp-config.php`).

= Как называть файлы изображений? =
По шаблону `background-{slug}.jpg`, где `{slug}` — это slug рубрики верхнего уровня. Для главной страницы — `background-home.jpg`.

= Что такое get_top_term()? =
Пользовательская функция, которая возвращает рубрику верхнего уровня для текущего поста. Должна быть добавлена в `functions.php` вашей темы или через отдельный плагин.

= Что происходит, если изображение для рубрики не найдено? =
Применяется только дефолтный цвет фона, указанный в настройках плагина.
